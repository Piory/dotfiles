# 品質保証ルール
- 必ず日本語で回答すること
- ハルシネーション禁止：事実確認できない内容を断言しないこと
- 不明な場合は「わかりません」と答える
- 推測の場合は必ず「（推測です）」と明記する
- 明確な指示がわからない場合は質問して確認すること。憶測で作業を進めないこと
- ユーザーに質問するときは `question` を使用すること
- コミットやコミットメッセージを生成する際は、それだけに集中し、他もファイルを変更しないこと
- コミットメッセージに `Co-Authored-By` や `🤖 Generated with ...` などの AI 署名・免責文を入れないこと（本文・トレーラーとも）
- セキュリティに関する内容は特に慎重に扱い、誤情報を提供しないこと
- 余計な作業は行わないこと。指示された内容にのみ集中すること
- 認知していない変更があった場合は、元に戻さずに意図的な変更か必ずユーザーへ確認すること
- Pythonなどのコード実行を禁止します。ただし、`.agents/skills` 内の Python ファイル実行は許可します。
- 指示があるまで PR の作成は行わないこと
- ユーザーの明示的な指示がない限り、勝手に push しないこと
- ライブラリの使い方を確認する際は find-docs スキルを使用すること
- テストを作成する場合は、必ず `単体テスト設計・実装` スキルを確認すること
- 実装（コード変更）が完了したら、必ず `codex-review-loop` スキルを実行してレビューが pass するまで対応すること
- `codex-review-loop` が完了したら、依頼元（指示側）エージェントは現在のブランチを対象に、作業全体の最終確認として `/simplify`、続けて `/review` を実行すること
- 上記の `/simplify` と `/review` は、`codex-review-loop` を完遂したあとの「作業全体の最終確認」として実行するものであり、必ず依頼元（Claude）自身のスキルとして実行すること。Codex など実装担当エージェントへ simplify / review を依頼してはならない（codex-review-loop の途中工程と混同しない）
- スキルを実行・参照する際は、会話中に内容が更新されている可能性があるため、毎回ディスクから最新の SKILL.md を読み直してから手順に従うこと（コンテキスト内の古い記憶で進めない）
- アプリケーションの実装コードに直接影響しない文書・説明・エージェント向け指示のみの変更では、`codex-review-loop` の実施は不要とする。

# ファイルの参照ルール
- ファイルの参照や検索を行う際は Serena MCP を使用すること
- Serena MCP を使用する対象は、Codex が参照しているワークスペース内のファイルのみとすること
- ワークスペース外のファイル取得には Serena MCP を使用しないこと
- スキル関連の取得には Serena MCP を使用しないこと
