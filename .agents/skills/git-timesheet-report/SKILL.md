# Git 勤怠作成スキル

## このスキルを適用するトリガー
次の依頼が来たら、このスキルを使う。
- 「今週の作業時間を勤怠にしたい」
- 「Git から勤怠時間を出して」
- 「今月の作業時間をいい感じにまとめて」
- 「コミット履歴から開始・終了を出して」

## 目的
- Git から確認できる事実を取得する。
- 勤怠入力に必要な開始・終了・休憩・実働を整形する。
- 事実と推測を混同しない。

## 基本方針
- まず `git config user.email` を取得し、そのメールアドレスを `--author` に使う。
- 期間内のコミットから、日別の最初と最後の時刻を集計する。
- Git に存在しない実績を事実として断定しない。
- ユーザー指定の勤務条件がある場合は、それを優先する。
- ユーザー指定がない場合の定時は `10:00-19:00`、休憩は `13:00-14:00`、実働は `8:00` とする。
- 推測を含む値は必ず「推測です」と明示する。
- 勤怠案の開始・終了・実働は分単位で出し、秒は表示しない。

## 必須手順
1. 対象期間を確定する。
2. 勤怠ルールを確認する。
   - 例: `10:00-19:00`
   - 休憩: `1時間`
   - 最低勤務時間: `8時間`
   - 丸め: `15分刻み`
   - コミットがない日の扱い
   - ユーザー指定がなければ、定時は `10:00-19:00`、休憩は `13:00-14:00`、実働は `8:00` として扱う。
3. `git log` で対象期間・対象作者のコミット時刻を取得する。
4. 日別に以下を分けて整理する。
   - Git 実測: 最初のコミット時刻 / 最後のコミット時刻
   - 勤怠案: 開始 / 終了 / 休憩 / 実働
5. 勤怠案は以下の順で作る。
   - 基本勤務時間を優先する
   - ただし最後のコミットが勤務時間外なら、分単位に丸めたうえで矛盾しないよう終了を延長する
   - 最低勤務時間を下回る場合は、その条件を満たすよう調整する
   - コミットがない平日は、Git 根拠がないことを明記した上で、ユーザー指定ルールまたはデフォルト定時に従って補完する
   - コミットがある日の実測範囲だけを合計して終わらず、補完した平日も合計に含める
6. 合計時間を計算する。
7. ユーザーが作業内容も求めた場合は、コミットメッセージや PR 内容から簡潔な業務内容案を作成する。
   - Git 根拠がない日の作業内容は推測であることを明記する。
   - 表現は短くする。例: `ブランド CRUD・ロゴ機能の実装`
8. Excel などの勤怠ファイルへ転記する場合は、対象列の形式に合わせて入力する。
   - `稼働時間詳細` のような列には `時間帯 作業内容` の形で入力する。
   - 秒は記載しない。
   - 合計値は分単位に合わせる。
   - `.xlsx` を編集した場合は `unzip -t` などで破損確認を行う。

## 出力ルール
- 事実と推測を分けて書く。
- 日別に `Git実績` と `勤怠案` を並べる。
- コミットがない日は、その旨を明記する。
- コミットがない平日は、定時補完した勤怠案を表示し、合計にも含める。
- 勤怠案では秒を表示しない。
- 合計は `勤務合計` `休憩合計` `実働合計` の形式で出す。
- 作業内容案を出す場合は、1日1行で簡潔に書く。

## 禁止事項
- Git に存在しない作業実績を、事実として書かない。
- 作者の特定を推測で行わない。
- コミットがない日を、Git 根拠ありと誤認させる表現をしない。
- 勤怠案に秒まで含めない。
- コミットがある日の最初〜最後の時刻だけを合計し、コミットがない平日を落とさない。

## コマンド例
```bash
AUTHOR_EMAIL=$(git config user.email)
git log --reverse \
  --since='2026-03-02 00:00:00' \
  --until='2026-03-07 00:00:00' \
  --author="$AUTHOR_EMAIL" \
  --pretty=format:'%ad' \
  --date=format-local:'%Y-%m-%d %H:%M:%S'
```

## 出力例
```text
- 2026-03-02 Git実績 18:03:20 - 21:43:58 / 勤怠案 10:00 - 21:43 / 休憩 13:00 - 14:00 / 実働 10:43
- 2026-03-04 Git実績なし / 勤怠案 10:00 - 19:00 / 休憩 13:00 - 14:00 / 実働 8:00（推測です）
```

## 作業内容例
```text
- 2026-06-01: ブランド管理機能の設計・調査（推測です）
- 2026-06-02: ブランド CRUD・ロゴ機能の実装
- 2026-06-03: 階層表示・検索機能の実装
```
