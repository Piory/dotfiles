# Git 勤怠作成スキル

## このスキルを適用するトリガー
次の依頼が来たら、このスキルを使う。
- 「今週の作業時間を勤怠にしたい」
- 「Git から勤怠時間を出して」
- 「今月の作業時間をいい感じにまとめて」
- 「コミット履歴から開始・終了を出して」

## 目的
- Git から確認できる事実を取得する。
- 勤怠入力に必要な開始・終了・休憩・実働を整形する。
- 事実と推測を混同しない。

## 基本方針
- まず `git config user.email` を取得し、そのメールアドレスを `--author` に使う。
- 期間内のコミットから、日別の最初と最後の時刻を集計する。
- Git に存在しない実績を事実として断定しない。
- ユーザー指定の勤務条件がある場合は、それを優先する。
- 推測を含む値は必ず「推測です」と明示する。

## 必須手順
1. 対象期間を確定する。
2. 勤怠ルールを確認する。
   - 例: `10:00-19:00`
   - 休憩: `1時間`
   - 最低勤務時間: `8時間`
   - 丸め: `15分刻み`
   - コミットがない日の扱い
3. `git log` で対象期間・対象作者のコミット時刻を取得する。
4. 日別に以下を分けて整理する。
   - Git 実測: 最初のコミット時刻 / 最後のコミット時刻
   - 勤怠案: 開始 / 終了 / 休憩 / 実働
5. 勤怠案は以下の順で作る。
   - 基本勤務時間を優先する
   - ただし最後のコミットが勤務時間外なら、矛盾しないよう終了を延長する
   - 最低勤務時間を下回る場合は、その条件を満たすよう調整する
   - コミットがない日は、Git 根拠がないことを明記した上で、ユーザー指定ルールに従って補完する
6. 合計時間を計算する。

## 出力ルール
- 事実と推測を分けて書く。
- 日別に `Git実績` と `勤怠案` を並べる。
- コミットがない日は、その旨を明記する。
- 合計は `勤務合計` `休憩合計` `実働合計` の形式で出す。

## 禁止事項
- Git に存在しない作業実績を、事実として書かない。
- 作者の特定を推測で行わない。
- コミットがない日を、Git 根拠ありと誤認させる表現をしない。

## コマンド例
```bash
AUTHOR_EMAIL=$(git config user.email)
git log --reverse \
  --since='2026-03-02 00:00:00' \
  --until='2026-03-07 00:00:00' \
  --author="$AUTHOR_EMAIL" \
  --pretty=format:'%ad' \
  --date=format-local:'%Y-%m-%d %H:%M:%S'
```

## 出力例
```text
- 2026-03-02 Git実績 18:03:20 - 21:43:58 / 勤怠案 10:00 - 21:45 / 休憩 13:00 - 14:00 / 実働 10:45
- 2026-03-04 Git実績なし / 勤怠案 10:00 - 19:00 / 休憩 13:00 - 14:00 / 実働 8:00（推測です）
```
